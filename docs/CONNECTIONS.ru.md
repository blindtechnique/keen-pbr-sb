# Меню «Соединения»

Страница показывает текущие и недавно замеченные TCP/UDP-соединения клиентских устройств Keenetic: состояние, протокол, адреса и порты, имя устройства и маршрут (`direct`, VLESS, AmneziaWG или другой outbound).

## Источники данных

- `/proc/net/nf_conntrack` — адреса, порты, протокол и состояние соединения;
- `fwmark`/`ctmark` и конфигурация keen-pbr — сопоставление с outbound;
- NDMS/RCI — имя клиентского устройства по IP;
- временный журнал запросов dnsmasq — best-effort сопоставление IP назначения с доменом.

Conntrack хранит только IP-адреса и не знает исходный домен. Поэтому имя домена показывается лишь тогда, когда его DNS-запрос видел dnsmasq keen-pbr. DoH/DoT на клиенте не наблюдается, а один CDN-адрес может соответствовать нескольким доменам; в этом случае интерфейс показывает наиболее свежий и дополнительные варианты.

## Хранение и ограничения

- история ограничена последними 1500 уникальными соединениями, а не временем;
- данные находятся только в оперативной памяти и исчезают после перезапуска;
- DNS-журнал находится в `/tmp`, ограничивается 2 МиБ и не пишет историю во flash;
- Web UI обновляется автоматически, поддерживает поиск, фильтрацию и сортировку.

Таким образом, нагрузка и расход памяти имеют жёсткую верхнюю границу, независимо от времени работы роутера.

## API

Текущая реализация использует `GET /api/connections`. Ответ не содержит реквизитов transport-manager или секретов конфигурации sing-box.
