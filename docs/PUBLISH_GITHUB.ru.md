# Первая публикация на GitHub

В этой версии GitHub Actions не используются: пакет для Keenetic AArch64 уже собран, а файлы `.github/workflows` намеренно отсутствуют.

1. Распакуйте подготовленный архив в корень репозитория `blindtechnique/keen-pbr-sb`. Файл `README.md` должен находиться в корне.
2. Добавьте исходники в Git, создайте коммит и отправьте ветку `main`.
3. Откройте **Releases → Draft a new release** и создайте тег, например `v3.0.7-sb.3`.
4. Прикрепите к Release готовые файлы:

   ```text
   keen-pbr_3.0.7-3_keenetic_aarch64-3.10.ipk
   SHA256SUMS
   ```

5. Опубликуйте Release и убедитесь, что GitHub помечает его как `Latest`.

После этого заработает однострочная установка из `README.md`: установщик берёт IPK и контрольную сумму из `releases/latest`.

Готовый IPK рассчитан на Entware `aarch64-3.10`. Для другой архитектуры или ABI нужен отдельный пакет с соответствующим именем.
