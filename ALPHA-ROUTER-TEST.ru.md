# Проверка alpha-сборки на Keenetic

Alpha предназначена только для тестового роутера или для установки в момент,
когда есть физический доступ к устройству. Перед началом сохрани экспорт
конфигурации keen-pbr-sb и конфигурацию Keenetic.

## 1. Установка

1. Дождись зелёного завершения workflow ветки `alpha`.
2. Убедись, что компьютер видит роутер:

   ```powershell
   ssh -p 222 root@192.168.1.1
   ```

3. Закрой SSH-сеанс и запусти `install-alpha-build.bat`.
4. После сообщения `OK-INSTALLED` открой `http://192.168.1.1:12121` и выполни
   полное обновление страницы через `Ctrl+F5`.

Установщик скачивает только артефакт `keen-pbr-alpha-ipk` из ветки `alpha`,
проверяет `SHA256SUMS`, передаёт IPK по SSH и переустанавливает пакет без
замены текущих настроек dnsmasq.

## 2. Быстрая проверка после установки

В SSH выполни:

```sh
/opt/etc/init.d/S80keen-pbr status
pid="$(cat /opt/var/run/keen-pbr.pid 2>/dev/null)"; echo "PID=$pid"; kill -0 "$pid"
curl -fsS http://127.0.0.1:12121/api/health/service
tail -n 100 /opt/var/log/keen-pbr.log
```

Ожидаемый результат:

- сервис запущен;
- `kill -0` завершается без ошибки;
- API возвращает JSON;
- в конце журнала нет цикла аварийных перезапусков.

## 3. Проверка жизненного цикла

В веб-интерфейсе последовательно:

1. Останови keen-pbr-sb.
2. Проверь, что интерфейс показывает остановленное состояние и остаётся
   доступен.
3. Запусти сервис.
4. Перезапусти сервис.
5. После каждой операции открой журнал операции и общий журнал.

После запуска и перезапуска проверь:

```sh
curl -fsS http://127.0.0.1:12121/api/health/service
nslookup config-hash.keen.pbr 127.0.0.1
ip rule show
ip route show table all | grep -E 'keen|unreachable|default'
```

Не должно быть зависшей операции, stale PID, повторяющихся правил или
состояния «работает» при фактически остановленном процессе.

## 4. Проверка конфигурационной транзакции

1. Измени безопасный параметр и нажми «Сохранить».
2. Убедись, что черновик исчез только после успешного применения.
3. Перезапусти страницу и сервис - изменение должно сохраниться.
4. Создай заведомо некорректный черновик, например со ссылкой на
   отсутствующее исходящее соединение.
5. Сохранение должно завершиться ошибкой, а действующая маршрутизация и
   файл `/opt/etc/keen-pbr/config.json` не должны замениться некорректной
   версией.

## 5. Проверка маршрутизации

Проверь по одному правилу для:

- прямого выхода;
- нативного WG/AWG-интерфейса;
- sing-box транспорта, если он установлен;
- urltest/failover;
- DNS-правила по доменному списку.

Для каждого варианта проверь задержку, внешний IP и результат встроенной
проверки маршрута. Затем временно останови выбранный туннель и убедись, что
failover или kill-switch срабатывает без утечки через основной WAN.

## 6. Что приложить к сообщению об ошибке

- время возникновения ошибки;
- экспорт диагностики из интерфейса;
- последние 200 строк `/opt/var/log/keen-pbr.log`;
- действие, после которого возникла ошибка;
- сохраняется ли ошибка после одного штатного перезапуска сервиса.

Не сбрасывай Entware до выгрузки диагностики: после сброса исчезнут журнал и
состояние, необходимые для разбора причины.
