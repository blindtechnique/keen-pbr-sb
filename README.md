# keen-pbr-sb

Русскоязычный форк [maksimkurb/keen-pbr](https://github.com/maksimkurb/keen-pbr) для маршрутизаторов Keenetic/Netcraze с Entware и управляемыми интерфейсами sing-box.

Проект остаётся полностью открытым: исходный код форка, transport-manager, установщик и веб-интерфейс находятся в этом репозитории. Каталог `third_party` содержит только открытые зависимости с их лицензиями. Локальные каталоги `.cache`, `build` и `frontend/node_modules` не являются частью репозитория и не должны загружаться на GitHub.

> Это независимый проект, не связанный с Keenetic или Netcraze и не поддерживаемый ими официально.

## Отличия от оригинального keen-pbr

- отдельный transport-manager для создания и контроля sing-box TUN;
- импорт `vless://`, `vmess://`, `trojan://`, `ss://`, `hysteria2://`, `hy2://`, `tuic://`, `anytls://`, `socks://`, `http://` и `https://`;
- произвольный sing-box outbound в формате JSON, поэтому работа не ограничена VLESS/Reality;
- поиск существующего sing-box в стандартных каталогах и возможность выбрать свой путь;
- установка и обновление официального sing-box из интерактивного установщика, включая запуск через совместимый загрузчик Entware на старой glibc;
- по умолчанию устанавливается протестированная версия sing-box `1.13.14`; если опубликована более новая версия, установщик предлагает её отдельно и предупреждает, что совместимость ещё не проверялась;
- нативные интерфейсы Keenetic, WireGuard и AmneziaWG в общей схеме outbounds;
- создание привязанного outbound и групп `urltest` для failover;
- sing-box TUN со стеком gVisor и `auto_route: false`: таблицами маршрутизации владеет keen-pbr;
- автоматические правила FORWARD для управляемых TUN;
- настраиваемые bootstrap DNS для разрешения имени VPN-сервера до запуска туннеля;
- URL-списки в текстовом формате и бинарные наборы sing-box `.srs`;
- DNS Override для Keenetic, генерация dnsmasq-конфигурации и DNS-диагностика;
- страница «Соединения»: активные TCP/UDP-соединения, состояния, адреса, порты, маршрут, имена устройств Keenetic, фильтр, автообновление и история до двух часов в оперативной памяти;
- быстрая настройка при создании списка: необязательное одновременное создание правила маршрутизации и DNS-правила;
- экспорт и импорт списков keen-pbr-sb; экспорт и импорт правил маршрутизации с проверкой существующих outbounds и интерактивной перепривязкой отсутствующих тегов;
- необязательная установка официального `nfqws2-keenetic` и встроенная страница управления nfqws2 в общем дизайне keen-pbr-sb;
- редакторы конфигурации, стратегий, списков и Lua-скриптов nfqws2, просмотр журналов, управление службой, обновление пакета и проверка доступности сайтов;
- набор редактируемых стратегий nfqws2 из поставляемого архива, создание собственных стратегий и удаление любых стратегий;
- общий экспорт/импорт всех списков nfqws2 и создание дополнительных `.list`-файлов;
- необязательная авторизация веб-интерфейса с HttpOnly cookie-сессией на семь дней;
- диагностический отчёт из веб-интерфейса;
- интерактивные установщик и деинсталлятор.

## Поддерживаемые платформы

Основная протестированная цель — Keenetic с Entware `aarch64`. Сборка также предусмотрена для `armv7`, `mips`, `mipsel` и `x64`, но каждую модель и ABI желательно проверять отдельно. Прошивка должна поддерживать Entware, policy routing и netfilter.

## Установка одной командой

Сначала установите Entware в `/opt`, затем подключитесь к роутеру по SSH как `root`:

```sh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/blindtechnique/keen-pbr-sb/main/install.sh)"
```

Если нет `curl`:

```sh
sh -c "$(wget -qO- https://raw.githubusercontent.com/blindtechnique/keen-pbr-sb/main/install.sh)"
```

Установщик полностью русифицирован. Он определяет архитектуру Entware, загружает подходящий `.ipk` из последнего GitHub Release, проверяет `SHA256SUMS`, предлагает установить или выбрать sing-box, задать логин и пароль веб-интерфейса, настроить Keenetic DNS Override и при желании установить nfqws2. Перед принятием sing-box обязательно запускается проверка `sing-box version`: неработающий файл больше не переносится в конфигурацию. Повторный запуск этой же команды выполняет обновление с сохранением конфигурации и предлагает оставить существующие параметры авторизации.

Протестированная версия sing-box — `1.13.14`. Установщик всегда позволяет выбрать её. Наличие нового официального выпуска проверяется отдельно: новая версия показывается в меню с явным предупреждением о том, что она ещё не тестировалась с keen-pbr-sb.

Важно: одного исходного кода в ветке `main` недостаточно. В GitHub Release должны находиться собранные пакеты с именами вида:

```text
keen-pbr_<версия>_keenetic_aarch64-<abi>.ipk
SHA256SUMS
```

### Авторизация веб-интерфейса

Установщик создаёт `/opt/etc/keen-pbr/auth.json` с правами `0600`. Это собственная локальная учётная запись keen-pbr-sb: можно ввести такие же реквизиты, как у `root` Entware или администратора Keenetic, но они не проверяются через NDMS и не синхронизируются при смене пароля роутера. Безопаснее задать отдельный пароль.

После входа сервер выдаёт случайную HttpOnly/SameSite cookie-сессию сроком на семь дней. Кнопка «Выйти» завершает текущую сессию. Поскольку стандартный адрес `http://my.keenetic.net:12121` не использует TLS, не публикуйте этот порт в интернет; авторизация рассчитана на доверенную локальную сеть или доступ через защищённый VPN/HTTPS reverse proxy.

## Удаление

```sh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/blindtechnique/keen-pbr-sb/main/uninstall.sh)"
```

Деинсталлятор отдельно спрашивает об удалении конфигурации, sing-box, установленного этим проектом, DNS Override и пакета nfqws2. Конфигурация nfqws2 по умолчанию сохраняется; Entware и внешняя установка sing-box не удаляются.

## Быстрый старт

1. В разделе «Транспорты» добавьте нативный интерфейс или sing-box-подключение.
2. Для sing-box можно вставить share-link либо готовый outbound JSON.
3. При необходимости задайте bootstrap DNS IP-адресами, по одному в строке.
4. Запустите транспорт и создайте outbound для появившегося интерфейса. Если запуск не удался, интерфейс теперь показывает исходную ошибку sing-box, а не только код HTTP 400.
5. При создании списка включите «Создать правило маршрутизации» и/или «Создать DNS-правило», чтобы сохранить список и связанные правила одной операцией. Если требуется нестандартный порядок или дополнительные условия, оставьте флажки выключенными и настройте всё вручную.
6. Для резервирования создайте `urltest` из двух или более outbounds.

Доменные списки работают, когда клиент получает DNS через dnsmasq роутера. DoH/Secure DNS или запущенный на клиенте VPN могут обойти правила роутера.

### Бинарные списки SRS

В поле URL списка можно указать прямую ссылку на `.srs`, например:

```text
https://raw.githubusercontent.com/SagerNet/sing-geosite/rule-set/geosite-category-ai-!cn.srs
```

keen-pbr скачивает набор и вызывает `sing-box rule-set decompile`. Используется тот же проверенный путь к sing-box, который записал установщик. Поддерживаются точные домены, суффиксы доменов и IP CIDR. Regex, keyword, source CIDR и инвертированные правила нельзя без потерь представить в dnsmasq/ipset, поэтому они пропускаются; для SRS требуется запускаемый sing-box 1.10 или новее.

### Имена устройств в «Соединениях»

Имена сначала читаются из lease-файлов dnsmasq, затем дополняются данными NDMS командой `show ip dhcp bindings`. Для зарегистрированных клиентов приоритет имеет заданное пользователем имя Keenetic, затем DHCP hostname. Устройство без регистрации и hostname будет показано только по IP.

## Импорт и экспорт

На страницах «Списки» и «Правила маршрутизации» находятся кнопки экспорта и импорта JSON. При импорте списки можно заменить полностью или объединить с существующими. При импорте правил проверяются теги outbounds: если нужного outbound нет, интерфейс просит выбрать замену из уже созданных outbounds. Это предотвращает сохранение заведомо неработающих правил.

## nfqws2

Установка nfqws2 необязательна. Русский установщик keen-pbr-sb предлагает подключить официальный репозиторий `nfqws/nfqws2-keenetic` и установить пакет. Если пользователь отказался, пункт «nfqws2» остаётся в меню и показывает команды установки через наш установщик или оригинальный репозиторий.

Встроенная страница заменяет отдельный PHP/lighttpd-интерфейс и использует общую авторизацию keen-pbr-sb. Перенесены функции оригинального `nfqws-keenetic-web`:

- запуск, остановка, перезапуск и reload службы;
- отображение версии и обновление пакета;
- структурированное редактирование `nfqws2.conf`;
- создание, редактирование и удаление списков;
- редактирование Lua-файлов, просмотр и очистка журналов;
- проверка HTTP-доступности сайта с маршрутизатора.

Дополнительно реализованы редактируемые предустановленные стратегии, пользовательские стратегии, применение стратегии к основному конфигу и пакетный импорт/экспорт всех списков. Конфигурация nfqws2 хранится в `/opt/etc/nfqws2`, пользовательские стратегии — в `/opt/etc/keen-pbr/nfqws-strategies`, встроенные пресеты — в `/opt/usr/share/keen-pbr/nfqws-strategies`.

Интеграция основана на проектах [nfqws2-keenetic](https://github.com/nfqws/nfqws2-keenetic) и [nfqws-keenetic-web](https://github.com/nfqws/nfqws-keenetic-web), распространяемых по лицензии MIT. Сам пакет nfqws2 устанавливается из официального репозитория проекта и не встраивается в IPK keen-pbr-sb.

## Сборка и публикация

Инструкции по первой публикации находятся в [docs/PUBLISH_GITHUB.ru.md](docs/PUBLISH_GITHUB.ru.md), по синхронизации с оригинальным проектом — в [docs/FORK_MAINTENANCE.ru.md](docs/FORK_MAINTENANCE.ru.md).

Готовый комплект публикуется без GitHub Actions и без файлов `.github/workflows`: исходники загружаются в ветку `main`, а заранее собранный IPK и `SHA256SUMS` вручную прикрепляются к GitHub Release.

```sh
make test
make package-keenetic-all
```

Не добавляйте в Git `.cache`, `build`, `frontend/node_modules` и готовые архивы. Пакеты `.ipk` публикуются как assets отдельного GitHub Release.
